## What Is This?
Programming is the process of designing, writing, testing, and maintaining the instructions that a computer follows to perform a specific task. A simple analogy for programming is writing a recipe for your favorite dish. Just as a recipe provides a step-by-step guide for a cook to prepare a meal, a program provides a set of instructions for a computer to execute a task. This matters to you because understanding what programming is will help you create interactive coding challenges for your Personal Computer Museum project.

## How It Works Internally
### What is a Program?
A program is a set of instructions that a computer follows to perform a specific task. These instructions are written in a programming language, which is a set of rules and syntax that the computer can understand.

### Source Code vs Machine Code vs Bytecode
Source code is the human-readable version of a program, written in a programming language. Machine code is the binary version of a program that the computer's processor can execute directly. Bytecode is an intermediate representation of a program that is generated by a compiler and executed by a virtual machine.

### Compilation vs Interpretation
Compilation is the process of translating source code into machine code before executing it. Interpretation is the process of translating source code into machine code line by line, executing each line immediately. Java is a unique case, as it is compiled to bytecode, which is then interpreted or JIT-compiled by the Java Virtual Machine (JVM).

### Java's Compile Pipeline
The Java compile pipeline involves the following steps:
- The Java compiler (`javac`) translates the Java source code into bytecode.
- The bytecode is stored in `.class` files.
- The `.class` files are executed by the JVM.

### What is Bytecode?
Bytecode is the intermediate representation of a Java program that is generated by the Java compiler and executed by the JVM. It is not machine code, but rather a platform-independent representation of the program that can be executed by the JVM on any platform.

### Platform Independence
Java's platform independence is achieved through the use of bytecode and the JVM. The JVM translates the bytecode into machine code for the specific platform, allowing Java programs to run on any platform that has a JVM.

### JIT Compilation
JIT (Just-In-Time) compilation is the process of translating bytecode into machine code at runtime. This process is performed by the JVM and allows Java programs to achieve performance close to native code.

### What is an Algorithm?
An algorithm is a step-by-step solution to a problem. It is a set of instructions that a computer follows to solve a specific task.

### Pseudocode
Pseudocode is a way of writing logic before writing code. It is a high-level representation of a program that is easier to read and understand than actual code.

### High-Level vs Low-Level Languages
Java is a high-level language, meaning it abstracts away many low-level details and provides a more convenient and efficient way of programming. C is a lower-level language, meaning it provides more direct access to hardware resources and requires more manual memory management.

### Why Java?
Java is a popular choice for programming because it is enterprise-grade, typed, battle-tested, and has a massive ecosystem for full-stack applications. It is also platform-independent, making it a great choice for developing cross-platform applications.

## Common Mistakes Beginners Make
**Wrong idea: Programming is only about writing code.**
Correct idea: Programming is about designing, writing, testing, and maintaining code.
**Looks right but is silently wrong:** Using a low-level language when a high-level language would be more suitable.
**Seems optional but critical at scale:** Not using platform-independent code, which can lead to compatibility issues.
**Missed config or flag:** Not configuring the JVM correctly, which can lead to performance issues.
**Interview question:** What is the difference between compilation and interpretation?
